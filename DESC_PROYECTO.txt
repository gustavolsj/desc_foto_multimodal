Este proyecto es una app frontend de React para catalogar fotos históricas con Gemini. No depende de un backend propio para funcionar en local o en hosting estático.

**Cómo está construido**
1. Stack base:
- React 19 + TypeScript + Vite, definido en package.json.
- Entrada HTML en index.html.
- Bootstrap de React en main.tsx.

2. Build y dev server:
- Desarrollo: script dev en package.json levanta Vite en puerto 3000 y host 0.0.0.0.
- Build: script build en package.json genera dist con Vite.
- Preview: script preview en package.json sirve el build.
- Configuración de bundler en vite.config.ts:
  - Plugins de React y Tailwind.
  - base: ./ para despliegue estático.
  - Alias @ hacia raíz del proyecto.
  - Inyección de process.env.GEMINI_API_KEY desde variables de entorno al frontend.

3. Estilos y UI:
- Tailwind v4 + variables de tema en index.css.
- Componentes UI tipo shadcn/base-ui en ui.
- Utilidad de clases cn en utils.ts.
- Config de componentes en components.json.

**Cómo funciona en runtime**
1. Inicio de app:
- index.html carga main.tsx.
- main.tsx renderiza App dentro de StrictMode.
- Toda la lógica funcional vive en App.tsx.

2. Gestión de API key:
- En App.tsx, la clave se toma así:
  - Primero localStorage (GEMINI_API_KEY guardada por usuario).
  - Si no existe, usa process.env.GEMINI_API_KEY inyectada por Vite.
- El modal de configuración permite guardar una clave en localStorage y recargar.

3. Carga de imágenes:
- Soporta:
  - Archivos sueltos.
  - Carpetas completas (input con webkitdirectory).
  - URL remota (con fetch; puede fallar por CORS).
- Cada imagen se convierte a Data URL base64 y se guarda en estado como ImageItem con:
  - id, url, nombre, resultado, estado de análisis, error.

4. Análisis con IA:
- Al analizar una imagen, App.tsx llama GoogleGenAI con:
  - modelo gemini-3-flash-preview.
  - imagen en inlineData.
  - prompt en español orientado a archivística histórica.
  - herramienta googleSearch habilitada.
  - responseSchema tipado para forzar JSON con campos concretos.
- El JSON devuelto se parsea y se guarda en result del item.
- También existe análisis por lote: recorre la cola y analiza pendientes uno por uno.

5. Visualización de resultados:
- Panel derecho muestra:
  - título, época, autor, ubicación, confianza.
  - descripción histórica.
  - justificación académica renderizada en Markdown (ReactMarkdown).
  - descripción de soporte material.
- Botón Maps abre Google Maps con coordenadas estimadas.

6. Exportación:
- Exporta el item actual a CSV (metadatos tipo Dublin Core + campos IA), generando y descargando un Blob localmente.

**Arquitectura funcional resumida**
- Capa de presentación: componentes UI en ui.
- Capa de estado y lógica: un único componente contenedor en App.tsx.
- Capa de IA: cliente directo a Gemini desde navegador (sin API intermedia propia).

**Notas importantes**
1. Es frontend-first:
- Aunque hay dependencias como express/dotenv en package.json, en el código actual no hay servidor ejecutándose para análisis.

2. Seguridad:
- Si se usa variable de entorno en build para GEMINI_API_KEY, al ser frontend puede quedar expuesta en el bundle.
- El flujo preferido aquí es clave por usuario en localStorage, pensado para despliegue estático.

3. README:
- Flujo básico de ejecución descrito en README.md: instalar dependencias, definir clave, correr dev.
